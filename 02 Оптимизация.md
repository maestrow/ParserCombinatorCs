## Контейнер, который ничего не делает заменять на его содержимое. 

Это опциональная задача. Польза от реализации пока сомнительна

При этом если у контейнера было имя и описание, то присвоить их у новосозданнуму парсеру.

Напомним, что такие контейнеры - это лишь средство удобного определения правил:

	var z = new Rule("my rule");
	...
	a.Expr = ...
	...
	z.Expr = a + b | Parser.Char('c') + d

Мы можем т.о. определять именованные правила. Смысл именованных правил только в удобстве отладки.
Однако т.о. при определении правила создается контейнер, который сам по себе ничего не делает (Expr).

Дерево отладочной информации:

	sql
	  or
	    select
	      and
	        SELECT
	        fields
	        FROM
	        table
	    insert

В дереве отладочной информации удобно пропускать парсеры, которые просто исполняют свой контейнер, более не делая ничего (таковыми являются правила). 

	sql or
	  select and
	    SELECT
	    fields
	    FROM
	    table
	  insert
